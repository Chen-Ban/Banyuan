# ADR-006: 采用 AGPL-3.0 + 商业双授权模式

**状态**: 已采纳
**日期**: 2026-05-14
**决策者**: 陈班

## 背景

Banyuan 既想作为开源项目获取社区贡献和影响力，又希望保留商业化空间。需要选择一种许可模式平衡两者。

## 决策

采用双重授权（Dual License）：

- **开源版本**：AGPL-3.0 — 要求通过网络提供服务的应用若使用了 Banyuan 代码，须同样以 AGPL-3.0 开源
- **商业授权**：闭源使用权，无需开源自身代码，无 AGPL 网络传播义务

## 理由

AGPL-3.0 的"网络传播条款"（copyleft 覆盖 SaaS）是关键：企业如果将 BanvasGL/XiangDi 集成到自己的 SaaS 产品中，必须开源整个产品 —— 大多数企业不会接受，自然会购买商业授权。而个人开发者、学术研究、开源项目不受影响，可以自由使用。

这是 MongoDB（SSPL）、GitLab（MIT → Ultimate 商业）、Grafana（AGPL）等项目验证过的模式。

## 后果

- 个人和开源社区可以自由使用、修改、贡献
- 企业客户需付费获取商业授权
- 所有贡献者需签署 CLA（贡献者许可协议），确保项目方有权进行双授权
- 代码仓库中需同时包含 LICENSE（AGPL-3.0）和 LICENSE-COMMERCIAL 文件
