## 套餐与权限的关联协议

**未实施**

套餐（Plan）是连接支付、权限、配额三个子系统的枢纽数据结构。一个 Plan 定义：包含哪些 Permission、AI 配额上限（月度 token / 月度次数）、价格。用户购买 Plan 后，系统同时写入权限表和配额表。

**决策链：** 三个子系统各自独立但需要一个协调点 -> Plan 作为声明式配置定义"这个等级的用户能做什么、能用多少" -> 避免三个子系统之间直接耦合。

**约束：**

- Plan 变更（升级/降级）立即生效权限，配额按剩余周期比例调整
- Plan 数据存储在独立的 Plans 集合，可热更新无需重启服务
- 自定义企业套餐通过管理后台配置，不需要代码变更
