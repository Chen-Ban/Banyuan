# Specs 索引

方案文档（Specification Documents）是决策的落地实施指南。与 ADR 的关系：

```
ADR（决策）── 定义"做什么、为什么"
  └── Spec（方案）── 指导"怎么做、细节是什么"
```

方案文档按与 ADR 相同的 5 个域分目录存放，文件平铺在各域目录下。每个方案文件由对应的 ADR 决策条目引用路由到达，不直接浏览。

---

## 目录结构

```
specs/
├── engine/      # 引擎核心实施方案
├── agent/       # AI 智能体实施方案
├── app/         # 应用平台实施方案
├── schema/      # 数据契约实施方案
└── product/     # 产品商业化实施方案
```

---

## 方案文件规范

每个方案文件应包含：

- **关联决策**：指明本方案对应 ADR 中的哪条决策（域/粒度/标题）
- **目标**：方案要达成的具体目标
- **实施步骤**：AI 可直接执行的分步指南
- **验收标准**：如何判断方案已正确落地
- **影响范围**：涉及哪些文件/模块

命名建议：`{简短主题}.md`，如 `layoutMode-grid.md`、`checkpoint-persistence.md`。
